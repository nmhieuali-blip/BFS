# GLOSSARY STANDARD

## Purpose

Standardize:
- Business terminology
- Technical terminology
- Healthcare terminology
- AI interpretation consistency

---

# Glossary Structure

Every glossary item must contain:

| Field | Description |
|---|---|
| Term | Official terminology |
| Meaning | Definition |
| Category | Business / Technical / Healthcare |
| Synonyms | Alternative naming |
| Forbidden Terms | Disallowed wording |
| Notes | Optional explanation |

---

# Naming Rules

## Use Singular Form

Preferred:
- Claim
- Appointment
- Invoice

Avoid:
- Claims
- Appointments

---

# Avoid Ambiguous Terms

Avoid:
- Data
- Process
- Transaction

Prefer:
- Insurance Claim
- Payment Transaction
- Patient Encounter

---

# Consistency Rules

## One Meaning Per Term

A term must not have multiple meanings.

---

## One Term Per Meaning

Avoid multiple terms for same concept.

Example:

Preferred:
- Encounter

Avoid:
- Visit
- Session
- Medical Visit

---

# Technical Terminology Rules

## API Naming

Use:
- Access Token
- Refresh Token
- API Gateway

Avoid:
- Session Token
- Main API

---

# Healthcare Terminology Rules

Must align with:
- HIS terminology
- Insurance terminology
- Hospital operation terminology

---

# Forbidden Terminology

Avoid:
- vague business wording
- local slang
- team-specific naming

---

# Example

| Field | Value |
|---|---|
| Term | Encounter |
| Meaning | Medical visit of patient |
| Category | Healthcare |
| Synonyms | Patient Visit |
| Forbidden Terms | Session |
| Notes | Used across all healthcare modules |

---

# AI Output Rules

AI-generated documents must:
- reuse glossary terms
- avoid undefined terminology
- follow approved naming convention

---

# Versioning

Glossary updates must include:
- version
- updated date
- updated by

Example:

v1.0
Updated: 2026-05-16
Updated By: BA Team
